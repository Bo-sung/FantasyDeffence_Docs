## 시스템 명칭

### Manor (장원/영지)

| 한글 | 영문 | 설명 |
|------|------|------|
| 영지 | Manor | 플레이어가 관리하는 영주의 영지/장원 전체 시스템 |
| 장원 | Manor | 영지와 동일 (중세 판타지 느낌 강조) |

**사용 예시**:
- 테이블명: `config_manor_building`, `manor_user`
- API: `/api/manor/load`, `/api/manor/building/place`
- 문서: "장원 시스템", "장원 데이터 로드"

**선택 이유**:
- 중세 판타지 세계관에 적합
- 영주성(castle)과 조합이 자연스러움
- "영주의 관리지"라는 의미가 명확

## 맵 관련 용어

### 기본 단위

| 한글 | 영문 | 설명 |
|------|------|------|
| 타일 | Tile | 1×1 최소 건설 단위 |

| 한글 | 영문 | 설명 |
|------|------|------|
| 영주성 | Castle | 4×4 타일, 중앙 고정, 이동/삭제 불가 |

## 데이터 저장소 계층

| 계층 | 용도 | 특징 | 예시 |
|------|------|------|------|
| **DB** | 영구 저장 | 유저 데이터 영속화, 트랜잭션 보장 | manor_user, manor_building_placement |
| **Redis** | 캐싱/상태 | 빠른 읽기, 임시 데이터, 실시간 상태 | 생산 진행 상태, 세션, Config 캐시 |

**데이터 흐름**:
- Config 데이터: DB 저장 → Redis 캐싱 → 클라이언트 로드
- 유저 데이터: DB 저장 ↔ Redis 캐싱 ↔ 실시간 업데이트

## 데이터베이스 명명 규칙

### Config 테이블 (설정 데이터)

**형식**: `config_manor_{entity}`

| 테이블명 | 설명 |
|----------|------|
| config_manor_building | 건물 마스터 데이터 |
| config_manor_building_level | 건물 레벨별 수치 |
| config_manor_castle_level | 영주성 레벨별 설정 |
| config_manor_terrain | 지형 데이터 |
| config_manor_global | 전역 설정 |

### 서버 DB 스키마

**형식**: `manor_{entity}` 또는 `manor_{category}_{entity}`

| 테이블명 | 설명 |
|----------|------|
| manor_user | 유저 장원 상태 |
| manor_unlocked_tiles | 유저별 해금된 타일 |
| manor_building_placement | 건물 배치 인스턴스 |
| manor_building_production | 생산 상태 |

## API 명명 규칙

### 기본 경로

```
/api/config/all          - Config 데이터 로드
/api/manor/load          - 장원 데이터 로드
/api/manor/building/*    - 건물 관련 API
/api/manor/area/*        - 영역 관련 API
```

### 주요 API

| API | 설명 |
|-----|------|
| GET /api/config/all | Config 데이터 일괄 로드 (로그인 시 1회) |
| GET /api/manor/load | 장원 진입 시 데이터 로드 |
| PUSH /manor/update | 장원 상태 변경 알림 (서버 푸시) |
| POST /api/manor/building/place | 건물 배치 |
| POST /api/manor/building/move | 건물 이동 |
| POST /api/manor/harvest | 자원 수확 |
| POST /api/manor/area/unlock | 영역 확장 (타일 해금) |
