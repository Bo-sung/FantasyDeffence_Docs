[← README로 돌아가기](../README.md)

## 개요

Fantasy Tower Defence의 게임플레이는 **3가지 핵심 컨텐츠**를 중심으로 설계되었습니다:

- **영지 시스템**: 게임의 **내실** 담당 (기반이 되는 메타 시스템)
- **스테이지 클리어**: 게임의 **중심** 담당 (실제 게임플레이, 성장 활용처)
- **캐릭터 수집**: 게임의 **유입** 담당 (주요 매출, 유저 유치 및 유지)

이 3가지는 메타 루프를 통해 유기적으로 연결되어 플레이어의 장기적 참여와 수익화를 동시에 달성합니다.

---

## 1️⃣ 영지 시스템 (Manor System)

### 정의
플레이어가 소유한 영지를 건설·업그레이드하고 자원을 생산하여 게임 진행의 기반을 마련하는 시스템

### 핵심 요소
- **건물 건설 & 업그레이드**: 대장간, 훈련소, 수리실, 생산 건물 등
- **자원 생산**: 영지 시설에서 자동으로 생산되는 자원 (은화, 식량, 마석, 보급, 인력)
- **수익화 포인트**: 건설 시간 가속, 슬롯 확장 등으로 과금 유도

### 플레이 흐름
```
건물 건설 → 자원 생산 → 자원 소비 (영웅 육성, 장비 강화) → 더 강한 영웅 육성 가능
```

### 역할
- **게임의 내실**: 게임 진행의 기반이 되는 메타 시스템
- **성장의 가시화**: 자원 생산 → 건물 업그레이드 → 더 많은 자원 생산 (선순환 구조)
- **일일 접속 유도**: 매일 자원을 수집하고 건설 완료를 확인하는 루틴 플레이
- **진행 속도 조절**: 과금으로 건설 시간 가속 → 유저 만족도와 수익화 양립

## 2️⃣ 스테이지 클리어 (Stage Clear)

### 정의
실시간 타워디펜스 전투를 통해 스테이지를 클리어하는 핵심 전투 컨텐츠

### 핵심 요소
- **실시간 전투**: 영웅 배치, 스킬 발동, 전술 변경으로 진행
- **준비 단계**: 영지에서 영입한 영웅 중 덱에 담긴 영웅들을 맵에 배치
- **웨이브 진행**: 점진적으로 강해지는 적들을 격퇴
- **보상 시스템**: 클리어 난이도에 따른 별 평가로 차등 보상

### 플레이 흐름
```
덱 편성 (메인) → 스테이지 진입 → 영웅/바리케이드 배치 (준비) →
웨이브 시작 → 실시간 전투 (스킬 발동, 영웅 이동 명령) →
클리어 or 재도전 → 보상 획득 → 영웅 수리
```

### 역할
- **성장 활용**: 영웅 육성과 링크 확장을 통한 성장을 실제로 활용하고 보상받는 공간
- **주력 컨텐츠**: 실제 게임플레이가 일어나는 곳으로, 플레이 시간의 대부분을 차지
- **전술적 재미**: 매 스테이지마다 다른 맵/웨이브 구성으로 매번 다른 전술 요구
- **보상 루프**: 클리어 → 보상 획득 → 영웅 육성 → 더 어려운 스테이지 도전의 선순환

## 3️⃣ 캐릭터 수집 (Character Collection)

### 정의
다양한 영웅을 영입하고 육성하여 더 강력한 팀을 구성하는 수집형 요소

### 핵심 요소

#### 1. 영웅 제조 (Free 경로)
- **자원 투입**: 영지에서 생산되는 자원 (식량, 마석, 보급, 인력) 투입
- **무료 획득**: 모집권만 소모, 대기 시간 필요
- **저격 가능**: 자원 비율을 조절하여 원하는 역할의 영웅 확률 증가

#### 2. 방문 영웅 영입 (Premium 경로 + Daily 접속 유도)
- **과금 직구**: 금화(결제 재화)로 즉시 확정 획득
- **일일 리셋**: 매 24시간마다 새로운 영웅 1명 방문
- **FOMO 활용**: 놓치면 언제 다시 올지 모르는 긴급성
- **필수 접속 이유**: 원하는 영웅을 놓치지 않으려면 매일 접속 필요

#### 3. 영웅 육성 & 링크
- **레벨링, 승급, 스킬 강화, 장비 장착**
- **중복 영웅 활용**: 링크 확장으로 부대 강화
- **등급 다양성**: 1성~5성 모든 영웅을 최종적으로 5성까지 육성 가능

### 플레이 흐름 (단기 & 장기)

#### 일일 플레이 (매일 반복)
```
로그인 → 방문 영웅 확인 (원하는 영웅인가?) →
원하면 금화로 즉시 구매 / 아니면 내일 기대 →
영웅 육성 재료 수집 → 스테이지 도전
```

#### 장기 플레이 (주/월 단위)
```
원하던 영웅 획득 → 육성 시작 (레벨/승급) → 링크 확장 →
강화된 팀으로 스테이지 클리어 → 더 많은 자원 획득 →
다음 목표 영웅 육성 (반복)
```

### 역할

#### 1. 주요 매출 수단
- **방문 영웅 직구**: 금화(과금 재화)로 원하는 영웅 즉시 획득
- **슬롯 확장**: 제조 슬롯 확장에 금화 소모
- **육성 가속**: 경험치, 강화 재료, 모집권 구매 등

#### 2. 일일 접속 유도 & FOMO 활용
- **방문 영웅 리셋**: 매 24시간마다 새로운 영웅 1명 등장
- **놓칠 위험**: 놓치면 언제 다시 올지 모르는 긴급성
- **필수 일일 접속**: 원하는 영웅을 잡기 위해 매일 게임 실행 필요

#### 3. 장기 수집 목표
- **전 영웅 5성 달성**: 명확한 장기 목표 제시
- **특정 영웅 완전 육성**: 육성할 영웅 선택의 자유도
- **지속적 플레이 동기**: 완전히 육성하기까지 수 개월 이상 소요

#### 4. 플레이 다양성
- **덱 구성의 자유도**: 어떤 영웅 조합인지에 따라 전술 변경
- **캐릭터 애착도**: 원하던 영웅을 얻었을 때의 만족감 극대화

## 3가지 컨텐츠 간 연결고리

### 역할 정의
- **영지 시스템**: 게임의 **내실** (기반 시스템)
- **스테이지 클리어**: 게임의 **중심** (주력 컨텐츠, 성장 활용처)
- **캐릭터 수집**: 게임의 **유입** (매출과 유저 유치의 중심)

### 메타 루프 (3~7일 단위)
```
┌──────────────────────────────────────────────────┐
│  [기반] 영지 시스템                               │
│  ↓                                               │
│  자원 생산 → 건물 업그레이드 (선순환)             │
│  ↓                                               │
│  [유입] 캐릭터 수집                               │
│  ↓                                               │
│  새로운 영웅 획득 → 육성 (과금 포인트)            │
│  ↓                                               │
│  [중심] 스테이지 클리어                           │
│  ↓                                               │
│  성장한 팀으로 더 어려운 스테이지 도전            │
│  ↓                                               │
│  보상 획득 → 영지 강화 (반복)                     │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 일일 루프 (1일 단위)
```
로그인 → [방문 영웅 확인] → 영지 자원 수집 → 스테이지 도전 3~5회 →
영웅 수리 → 로그아웃

* [방문 영웅]은 놓치면 24시간 후에 새로운 영웅으로 교체되므로
  원하는 영웅을 놓치지 않으려면 매일 접속 필요!
```

### 주간/월간 루프
```
신규 이벤트 스테이지 도전 → 신규 영웅 영입 → 기존 팀 강화 → 신규 도전 시도
```

## 개발 우선순위

| 우선순위 | 컨텐츠 | 이유 |
|---------|--------|------|
| 1순위 | **스테이지 클리어** | 실제 게임플레이가 일어나는 곳. 전투 로직, 밸런싱, 맵/웨이브 설계가 모두 영향. 게임의 재미 여부를 결정 |
| 2순위 | **캐릭터 수집** | 주요 매출 수단. 스테이지 클리어에 필요한 다양한 영웅 제공. 유저 유치 및 유지의 핵심 |
| 3순위 | **영지 시스템** | 게임의 내실과 선순환 구조. 스테이지 + 캐릭터 시스템 완성 후 추가 |

### Phase별 포함 여부

| 컨텐츠 | Phase 1 | Phase 2 | Phase 3+ |
|--------|---------|---------|----------|
| 스테이지 클리어 | ✅ 필수 | ✅ 확장 | ✅ 신규 맵 |
| 캐릭터 수집 | ✅ 필수 (20명) | ✅ 확장 (30명+) | ✅ 신규 영웅 |
| 영지 시스템 | ✅ 기본 | ✅ 고도화 | ✅ 신 건물 |
